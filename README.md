 𓆩⚜𓆪  Broken Object Level Authorization (BOLA) es una de las vulnerabilidades más críticas y frecuentes en aplicaciones web y APIs 𓆩⚜𓆪. Consiste en que la aplicación no verifica correctamente si un usuario tiene permisos para acceder, modificar o eliminar un objeto específico (como cuentas, pedidos, documentos o perfiles), permitiendo que un atacante manipule identificadores para acceder a recursos de otros usuarios. Detectarla implica comprender la lógica de autorización, analizar cómo se referencian los objetos y comprobar si el servidor valida adecuadamente la propiedad y los permisos sobre cada recurso solicitado.<br><br>
